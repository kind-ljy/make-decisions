# 开放性世界 AI 互动叙事游戏 · 设计文档

> 文档日期：2026-05-28
> 项目代号：暂定 **MakeDecisions**（与仓库同名）
> 形态：网页端单机互动叙事游戏（首版）

---

## 一、核心思想（One-liner）

> **"你给一个主题，AI 带你活一遍。"**

玩家用一句话描述自己想体验的世界（比如"末日废土的赏金猎人"、"晚清名医"、"火星殖民地工程师"），系统生成开场剧情和 3-4 个行动选项；玩家每做一次选择，AI 都会基于完整的故事记忆推进剧情、呈现新场景、给出新选项。**没有预设剧本，每一局都是独一无二的故事**。

## 二、设计哲学（三条铁律）

1. **主题完全开放**：不预设题材库，玩家输入即世界。系统应能驾驭从修仙到职场、从童话到悬疑的任意题材。
2. **选择必须有重量**：选项不是文字装饰，而是真正改变后续走向的分叉。同一玩家两次玩同一主题，体验应明显不同。
3. **故事有记忆**：AI 必须记住玩家做过的每一个关键选择、获得的物品、遇到的角色，避免"金鱼记忆"（前后矛盾是这类游戏的头号体验杀手）。

## 三、玩家旅程（Player Journey）

```
[首页]
  ↓ 玩家输入主题或选择灵感卡片
[世界生成中... (2-4s)]
  ↓ AI 返回开场叙事 + 3-4 个选项
[场景 1：阅读叙事 → 点击选项]
  ↓
[场景 2：基于上一选择生成的新叙事 + 新选项]
  ↓
[场景 N：循环……直到玩家结束 / 触发结局]
  ↓
[结局回顾：展示完整选择路径 + 关键转折点]
```

### 单局节奏目标
- 每段叙事文字：**150-300 字**（一屏读完，不滚屏）
- 选项数量：**3-4 个**（少则单调，多则选择疲劳）
- 单局长度：**10-30 个回合**（约 15-30 分钟）
- 响应等待：**2-5 秒**（用打字机效果掩盖延迟）

## 四、核心系统设计

### 4.1 故事状态机（StoryState）

每一局游戏在内存里维护一个故事状态对象，每次请求 LLM 时都把它压缩后塞进 prompt：

```typescript
interface StoryState {
  theme: string;              // 玩家输入的主题
  protagonist: {              // 主角设定（首回合 AI 生成）
    name: string;
    background: string;
    traits: string[];
  };
  worldSetting: string;       // 世界观设定（首回合 AI 生成，约 100 字）
  inventory: string[];        // 持有物品 / 关键道具
  relationships: {            // 与 NPC 的关系
    [npcName: string]: { description: string; affinity: number };
  };
  history: Array<{            // 关键选择历史（不是全部对话）
    turn: number;
    summary: string;          // AI 自己总结这一回合的关键事件
    choice: string;
  }>;
  currentScene: string;       // 当前场景描述
  turn: number;
}
```

> **关键技巧**：不要把所有历史对话原文都喂给 LLM（token 会爆炸）。只保留 AI 自己生成的"回合摘要"，这是控制成本和保持长期一致性的核心。

### 4.2 LLM 交互协议（结构化输出）

每次调用 LLM，强制要求返回 JSON：

```json
{
  "narrative": "夜雨敲打着青石板，你握紧腰间的短刀……（200 字叙事）",
  "scene_summary": "在雨夜街角遇到神秘黑衣人，对方递来一封信",
  "choices": [
    { "id": "A", "text": "拆开信件阅读", "hint": "谨慎" },
    { "id": "B", "text": "把信扔回去转身就走", "hint": "果断" },
    { "id": "C", "text": "假装答应，暗中跟踪对方", "hint": "心机" }
  ],
  "state_updates": {
    "inventory_add": ["神秘信件"],
    "relationship_changes": { "黑衣人": "+1 好奇" },
    "ending_triggered": false
  }
}
```

前端拿到后：
- 用打字机效果渲染 `narrative`
- 渲染 `choices` 为可点击按钮
- 把 `scene_summary` 和 `state_updates` 合并进 `StoryState`

### 4.3 Prompt 工程（系统提示词骨架）

```
你是一名"开放世界互动小说"的叙事引擎。

【硬性规则】
1. 始终用第二人称"你"叙述，让玩家代入主角
2. 每次输出严格遵循 JSON Schema（不许夹带任何解释文字）
3. narrative 控制在 150-300 字，画面感强、留悬念
4. 必须给出 3-4 个截然不同的选项，禁止"换汤不换药"
5. 必须基于"故事状态"保持连贯，不能凭空冒出未提及的人物/物品
6. 当玩家累计 15 回合后，开始铺垫结局（伏笔回收）
7. 当玩家累计 25 回合后，主动给出"导向结局"的关键选项

【主题】{theme}
【故事状态】{compressed_state_json}
【玩家上一选择】{last_choice}

请生成下一回合：
```

## 五、技术架构

```
┌─────────────────────────────────────────┐
│  浏览器（前端 SPA）                      │
│  - 主题输入页 / 叙事页 / 结局页          │
│  - 用 localStorage 保存进度（断点续玩）  │
└──────────────┬──────────────────────────┘
               │ HTTPS POST /api/turn
               ▼
┌─────────────────────────────────────────┐
│  Serverless 后端代理                     │
│  （Vercel / Cloudflare Workers，零成本） │
│  - 注入 API Key                          │
│  - Prompt 拼装                           │
│  - JSON 校验与错误重试                   │
│  - 限流（防止白嫖）                      │
└──────────────┬──────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────┐
│  智谱 GLM-4-Flash API（永久免费）        │
└─────────────────────────────────────────┘
```

### 5.1 前端技术选型
- **HTML + 原生 JS + Tailwind CSS**（首版极简，不引入 React，部署只是一个静态文件）
- 后续若复杂度上升，再迁移到 Vite + React
- 打字机效果：`stream` 模式 + `requestAnimationFrame`

### 5.2 后端技术选型
- **Cloudflare Workers**（免费 100,000 次请求/天，国内可访问）
- 单文件 `worker.js`，约 100 行代码
- 用环境变量存 `ZHIPU_API_KEY`

### 5.3 数据持久化（首版）
- 不要数据库，所有进度存浏览器 `localStorage`
- 玩家关掉网页也能续玩（自动加载上一局）
- 后续若要做"分享存档"功能，再加 Supabase / D1

## 六、风险与对策

| 风险 | 概率 | 对策 |
|------|------|------|
| LLM 输出不合规 JSON | 高 | 双重防御：prompt 强制 + 后端 JSON.parse 失败时重试一次 |
| 长游戏后剧情前后矛盾 | 高 | 仅保留摘要+关键状态，不全量传上下文；设上限 30 回合 |
| 玩家恶意刷接口耗 token | 中 | Worker 层做 IP 限流（每 IP 每天 100 次） |
| 玩家输入违规主题 | 中 | 智谱自带内容安全过滤；前端加敏感词预过滤 |
| 选项"假分叉"（点哪个剧情都一样） | 中 | Prompt 中明确要求"选项必须导向截然不同的后续"；可加自动评估 |
| 单次响应 4-8 秒，玩家觉得卡 | 高 | 用流式输出 + 打字机动画，让等待变成体验 |

## 七、首版（MVP）功能清单

**必须有（P0）：**
- [ ] 主题输入页 + 3 个示例主题快捷入口
- [ ] 叙事文本流式打字机渲染
- [ ] 3-4 个选项按钮交互
- [ ] StoryState 在前端维护并随每次请求传给后端
- [ ] Cloudflare Worker 代理 + 智谱 API 集成
- [ ] localStorage 自动存档/续玩
- [ ] 结局页（"你的故事到这里"+ 选择路径回顾）

**先不做（YAGNI）：**
- ❌ 用户系统/账号
- ❌ 多语言切换（先做中文）
- ❌ 图片/音效（首版纯文字）
- ❌ 多人协作叙事
- ❌ 分支可视化树
- ❌ 付费 / 排行榜
