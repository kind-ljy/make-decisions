- **关键技巧**：StoryState 状态机 + AI 生成的回合摘要（避免上下文爆炸）

## 2026-05-28 17:27 安全收尾指引

向用户解释了 `wrangler secret put` 的具体操作方式与完整安全收尾顺序：

### wrangler secret put 用法
- **交互式**：执行 `wrangler secret put ZHIPU_API_KEY` → 终端提示 `✔ Enter a secret value:` → 粘贴 Key（不回显）→ 回车
- **stdin 方式**：`echo -n "key" > /tmp/zk.txt && wrangler secret put ZHIPU_API_KEY < /tmp/zk.txt && rm /tmp/zk.txt`
- 看 secret 列表：`wrangler secret list`
- 删除：`wrangler secret delete ZHIPU_API_KEY`
- Worker 不存在时 secret put 会问是否创建，选 y 即可

### 推荐执行顺序（不可颠倒）
1. 智谱控制台生成新 Key（暂留旧 Key）
2. wrangler login → cd worker → wrangler secret put ZHIPU_API_KEY → 粘贴新 Key
3. wrangler deploy 部署 Worker
4. curl 验证 /api/health 与 /api/turn
5. 清空 index.html 的 DEFAULT_API_KEY（已完成 ✅）
6. 前端切到 Worker 模式，部署 Pages，收紧 ALLOWED_ORIGINS
7. 最后才去智谱控制台撤销旧 Key

## 2026-05-28 19:44 Deno Deploy 也走不通：海外节点回中国超时

部署成功，URL：`https://make-decisions-api.kind-ljy.deno.net`
- ✅ `/api/health` 国内 50ms 内返回（说明 Deno Deploy 国内可访问）
- ❌ `/api/turn` 和 `/api/image` 60 秒超时无响应
- 通过 response header 中的 `via: HTTP/2 ord.vultr.prod.deno-cluster.net` 定位 Deno 把你路由到了美国 Chicago 节点
- **根本原因**：Deno Worker 在美国 → 调智谱（北京）跨太平洋链路不稳，60s 都不返回

### 关键经验
**Deno Deploy / Vercel / 任何海外 Serverless** 都不适合代理国内 LLM API：
- 即使 Worker 域名国内可访问，Worker 出口在海外，调国内 API 超时
- 智谱 / DeepSeek / 通义 / 文心 等大陆 LLM 都有这个问题

### 推荐方案优先级
1. 腾讯云函数 SCF（国内、免费、与智谱同域）
2. 阿里云函数计算 FC
3. 自建 Cloudflare 自定义域（需买域名）
4. 本地直连模式（不部署后端，朋友玩自己填 Key）

### 当前实用路径
用户当前阶段：自己和朋友玩 → 用本地直连模式（前端 ⚙ 里关掉 Worker 开关）即可。
未来要正式上线 → 改腾讯云 SCF。
